# PROJECT — Square Lake Family Dentistry website

> Self-contained handoff context. This folder is the source of truth for "what is this project" —
> it travels with the git repo, so any device (or any assistant) can pick up the work without
> relying on external/account memory.

## What this is
A fast, responsive, multi-page **static marketing website** for Square Lake Family Dentistry,
a dental practice in Troy, MI (owner: Dr. Saif Hanna). Plain HTML/CSS/JS — **no runtime
dependencies** — deployed on **GitHub Pages**.

- **Live site:** https://zandstrading1-hash.github.io/square-lake-dentistry/
- **GitHub repo:** https://github.com/zandstrading1-hash/square-lake-dentistry (public)
- **Default branch:** `main`
- **Booking model:** phone-first — every primary CTA is "Call to book" → tel:(248) 558-2785.
  No online booking. The Contact page has an optional non-urgent message form only.

## Tech stack
- **Frontend:** hand-written HTML5, CSS3 (`styles.css` — design system + components), vanilla JS
  (`script.js` — mobile menu, sticky header, scroll reveal, contact-form handler).
- **Page generator (optional):** Python 3 — **standard library only, NO pip packages**. Inner
  pages are generated from small Python content modules so the shared header/footer stay in sync.
  `index.html` is hand-authored and is NOT generated.
- **Hosting:** GitHub Pages (legacy branch build: branch `main`, folder `/`).
- **No** package.json, no Node build, no framework, no database, no backend, **no secrets**.

## File map
- `index.html` — hand-authored homepage (custom hero). Not regenerated by build.py.
- Content pages: `about, services, new-patients, insurance, reviews, faq, contact, blog` (+ 3 posts),
  8 service pages, legal pages (`privacy, terms, accessibility, notice-of-privacy-practices`), `404.html`.
- `styles.css`, `script.js` — shared assets.
- `robots.txt`, `sitemap.xml` — generated SEO files.
- `build.py` — regenerates all generated pages + sitemap + robots.
- `buildlib.py` — shared <head>/header/footer/CTA helpers + page renderer.
- `content_core.py`, `content_services.py`, `content_blog.py`, `content_legal.py` — page content.
- `resync_index.py` — re-applies the shared header/footer to `index.html`.
- `patch_index.py` — one-off first-run homepage alignment (rarely needed).
- `README.md` — fuller readme (editing, accessibility, publishing).
- `.claude-handoff/` — this folder (cross-device context).

## Accessibility
Built to WCAG 2.1 AA: semantic landmarks, skip-to-content link, full keyboard nav, visible
`:focus-visible` indicators, AA color contrast, labeled forms, `prefers-reduced-motion` support,
and a published Accessibility Statement (`accessibility.html`).
