# Google Drive — what to sync and what NOT to sync

This project is set up to be worked on from multiple devices. **Read this before changing how it
syncs.**

## TL;DR
- The **code + history sync through GitHub** (`git clone` / `git pull` / `git push`) — not through Drive.
- Google Drive carries the **handoff context** (`.claude-handoff/`) and this note, so any device can
  get oriented fast.
- Keep secrets and generated/heavy folders (below) **out of any synced location**.

## Never sync — secrets / credentials
- `.env`, `.env.*`, anything containing API keys, tokens, or passwords
- `*.key`, `*.pem`, `*.pfx`, `id_rsa*`, service-account JSON
- private databases (`*.sqlite`, `*.db`), account / credential files

> This project currently contains **none** of these. Keep it that way — never commit or sync them.

## Never sync — generated / heavy (recreate per device)
- Virtual environments: `.venv/`, `venv/`, `env/`, `ENV/`
- `node_modules/`, `dist/`, `build/`, `.next/`, `.turbo/`
- Caches: `__pycache__/`, `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/`, `*.pyc`
- `logs/`, `*.log`, and any large raw-data folders
- OS / sync litter: `.DS_Store`, `Thumbs.db`, `desktop.ini`

## The important one: do NOT sync a live `.git` folder through Drive
Google Drive (and OneDrive / Dropbox) sync files independently and asynchronously. The `.git`
folder is many small interdependent files; if two devices are active, or a sync lands mid-commit,
the repository can be **corrupted** (conflicted-copy files inside `.git`, broken refs, lost objects).

**Use GitHub for git sync** (`git pull` / `git push`). Ideally keep your working git clone OUTSIDE
Drive/OneDrive (e.g. `C:\dev\` or `~/dev/`). Note: this repo currently lives under OneDrive on the
origin machine — that already carries this same risk, which is another reason the recommended
workflow is to clone fresh from GitHub on each device.

## If you choose to keep the whole repo in Drive anyway
It can work for a **solo** user, but only if you: (1) edit from **one device at a time**, (2) let
Drive finish syncing (green check) before switching devices, and (3) treat GitHub as the backup.
It is still riskier than git-based sync — prefer the GitHub workflow above.
