# Google Drive — sync notes (what to keep OUT, and how to stay safe)

This project lives in Google Drive (`My Drive/Projects/square-lake-dentistry`) and syncs across your
devices. Read this before adding new kinds of files.

## Setup chosen: whole project (incl. git history) in Drive
That's intentional — it gives you one folder you can open and edit on either device. It works for a
solo user **as long as** you follow the golden rules in `OPEN-ME-FIRST.md`:
- one device at a time,
- let Drive finish syncing (up to date) before switching devices,
- commit + push to GitHub often as a backup.

The risk you're managing: a file-syncer can corrupt a live `.git` folder if two devices write to it
at the same time. GitHub is your safety net.

## Never put these in this folder (or any synced folder)
**Secrets / credentials** — `.env`, `.env.*`, API keys, tokens, passwords, `*.key`, `*.pem`, `*.pfx`,
`id_rsa*`, service-account JSON, private databases (`*.sqlite`, `*.db`), account files.
> This project has none today. If you ever add them, keep them OUTSIDE Drive entirely.

**Generated / heavy folders** — recreate these per device instead of syncing them:
`.venv/`, `venv/`, `env/`, `node_modules/`, `dist/`, `build/`, `.next/`, `.turbo/`,
`__pycache__/`, `.pytest_cache/`, `.mypy_cache/`, `.ruff_cache/`, `*.pyc`, `logs/`, `*.log`,
and large raw-data folders.

> Google Drive can't selectively skip a subfolder, so the protection is simply: **don't create these
> here.** They're also in `.gitignore`, so git won't track them. This project currently has none.

## If it ever breaks (corrupted .git, weird "conflicted copy" files)
Your work is safe on GitHub. Delete the local copy and restore:
```
git clone https://github.com/zandstrading1-hash/square-lake-dentistry.git
```
